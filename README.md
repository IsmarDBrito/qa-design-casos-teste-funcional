
Objetivo
Demonstrar proficiência na fase de planejamento de testes, essencial para qualquer ciclo de desenvolvimento de software. O foco foi mapear requisitos e criar artefatos de teste claros e rastreáveis.

Habilidades de QA Demonstradas
Design de Casos de Teste: Criação de testes com etapas, resultados esperados e critérios de aprovação/falha.

Checklist Design: Elaboração de checklists abrangentes para validação de layout (UI) e fluxo (métodos de pagamento).

Mapeamento Funcional: Identificação de cenários de teste críticos (ex: botões e fluxo de locação).

Documentação: Uso de planilhas para documentação formal dos artefatos de teste.

Conteúdo do Repositório

casos_teste_locacao.csv: Casos de teste para o fluxo principal (locação/compra).

checklist_layout.csv: Checklist para validação de interface do usuário (UI).

casos_teste_botoes.csv: Casos de teste para verificação funcional de elementos interativos.
